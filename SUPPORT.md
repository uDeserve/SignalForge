# Support

## Best Entry Points

Use the GitHub repository for most product and integration discussions.

- bug reports: GitHub Issues
- feature requests: GitHub Issues
- design and product direction: GitHub Discussions or Issues, if enabled

## Before Asking For Help

Please check:

- `README.md`
- `docs/quick-start.md`
- `docs/github-app-setup.md`
- `docs/live-e2e.md`
- `AGENT_README.md`

## Good Support Requests

The fastest-to-resolve reports usually include:

- your install mode: preview, PAT, or GitHub App
- the commands you ran
- the relevant environment variables, redacted as needed
- what you expected to happen
- what actually happened

## Product Focus

SignalForge is optimized first for:

- small web application teams
- independent developers
- GitHub-native maintainer workflows
- coding-agent-friendly integration paths
